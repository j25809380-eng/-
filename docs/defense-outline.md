# FitNote 毕设答辩提纲

## 1. 选题背景

- 健身记录类产品在训练持续性、数据沉淀和目标管理上有明确场景价值
- 微信小程序具备低门槛传播和便捷使用优势，适合学生项目落地
- 课题目标是设计并实现一个兼具视觉表现、业务完整度和工程规范性的健身记录系统

## 2. 系统目标

- 面向普通健身用户，提供训练记录、计划管理、数据分析和社区互动
- 面向教师答辩展示，提供后台管理系统与数据大屏
- 面向工程实现，采用前后端分离、JWT 鉴权、RESTful API 和 MySQL 持久化

## 3. 技术架构

- 前端：微信小程序原生
- 后端：Spring Boot 3
- 数据库：MySQL 8
- 鉴权：JWT
- 管理端：静态后台页面 + 后端接口

## 4. 核心功能

- 用户系统：登录、资料编辑、头像上传、训练目标设置
- 训练记录：新增训练、历史记录、PR 统计
- 训练计划：系统计划、自定义计划、按日训练配置
- 数据分析：训练趋势、体重变化、月度报告
- 社区模块：发动态、点赞、评论
- 后台系统：用户管理、训练数据、社区审核、数据大屏

## 5. 特色与亮点

- 保留 Stitch 高保真 UI 风格，完成真实业务化开发
- 训练计划、自定义计划、数据分析和社区形成完整闭环
- 管理端和数据大屏提升了项目答辩展示效果
- 工程结构清晰，便于后续继续扩展

## 6. 难点与解决方案

- 小程序与后端联调：通过请求层封装和 mock 回退提升开发效率
- 权限控制：通过 JWT 区分普通用户与管理员角色
- 数据展示：通过月报、排行榜和大屏增强信息可视化
- 演示环境：通过 H2 demo 模式和 Docker MySQL 模式兼顾开发与答辩

## 7. 结论与展望

- 当前系统已具备完整的小程序业务链路和后台演示链路
- 后续可继续扩展真实微信登录、社区图片上传、AI 训练建议和更完整的数据图表
