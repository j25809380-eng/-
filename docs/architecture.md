# FitNote 项目整体架构设计

## 1. 项目定位

FitNote 面向健身爱好者，核心价值是帮助用户完成训练计划管理、训练记录追踪、个人数据分析与社区互动。项目形态参考“训记”，但视觉风格采用 Stitch 已生成的深色高质感方案，以适配毕业设计展示和商业化表达。

## 2. 技术架构选型

### 2.1 前端

- 方案：微信小程序原生
- 选择原因：
  - 当前已有 Stitch 导出的多张静态页面原型，直接迁移到原生小程序成本最低
  - 无需额外编译框架，微信开发者工具中可直接运行
  - 对毕业设计答辩更友好，结构清楚、可解释性强
  - 可以通过自定义组件实现页面复用与风格统一

### 2.2 后端

- 方案：Spring Boot 3 + Spring Security + JWT
- 特点：
  - 符合 Java 后端主流工程规范
  - 易于扩展为后台管理系统和权限体系
  - 易于对接 MySQL、对象存储、图表大屏和审核流程

### 2.3 数据层

- 生产方案：MySQL 8
- 演示方案：后端 `demo` 配置内置样例数据，方便快速运行
- 目标：
  - 核心训练与社区数据持久化
  - 支持统计分析与月度报告
  - 为后台管理系统预留审核、日志和大屏数据来源

## 3. 系统分层

### 3.1 小程序端分层

- `pages`：页面层，承接 Stitch UI 和页面交互
- `components`：通用组件，如底部导航、标题、统计卡片
- `utils`：请求、路由、格式化、mock 数据
- `styles`：全局主题变量、暗黑高级风格、阴影和渐变定义

### 3.2 后端分层

- `controller`：接口层，接收请求和返回统一响应
- `service`：业务层，处理用户、训练、计划、社区逻辑
- `repository/entity`：数据访问与实体建模
- `security`：JWT 认证与接口保护
- `bootstrap`：样例数据初始化

## 4. 功能模块拆分

### 4.1 用户系统

- 微信登录
- 用户资料编辑
- 身高、体重、目标设定
- 训练等级与个人统计卡片

### 4.2 训练记录

- 创建训练记录
- 记录动作、重量、次数、组数、备注
- 自动带出上次训练重量
- 训练完成页与 PR 记录

### 4.3 训练计划

- 推荐计划
- 自定义计划
- 计划按天展开
- 打卡完成

### 4.4 数据分析

- 周训练频率
- 训练量趋势
- 体重变化
- PR 统计
- 月度概览

### 4.5 社区

- 动态流
- 点赞、评论、收藏
- 分享训练成果
- 排行榜

### 4.6 AI 助手

- 训练建议
- 恢复建议
- 饮食建议
- 历史记录问答入口

### 4.7 后台管理系统

- 用户管理
- 训练数据管理
- 社区内容审核
- 数据可视化大屏

## 6. 前后端通信方式

- 协议：HTTP/JSON
- 风格：RESTful
- 鉴权：`Authorization: Bearer <token>`
- 返回格式统一：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 7. 第一阶段目标

- 完成高保真页面落地为可运行小程序
- 打通用户、计划、动作、训练、社区、分析等基础接口
- 提供 `demo` 直跑模式
- 为第二阶段接入 MySQL 实库和后台管理端做好结构准备
